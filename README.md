# Harper Subjectivity Rendering Skill

When Harper sends a customer an email about their policy, the underwriter's raw subjectivities (the carrier's list of conditions attached to a quote) must never appear as-is. That raw text mixes real customer asks with payment references, bind instructions, carrier rescind rights, and underwriter margin notes. This skill is the company standard for turning that raw list into customer-safe copy: classification is responsibility-based (who acts: the client provides, the client signs via DocuSign, or the broker and carrier handle it), and everything that does not positively prove customer ownership is hidden by default. `SKILL.md` contains the full decision tree, a labeled example bank drawn from real production carrier language, the exact rendering contracts (loss runs, the anonymous DocuSign line, the generic confirmation line), and the process guardrails that keep the rules from regressing.

To install for a Cursor agent, copy `SKILL.md` into `~/.cursor/skills/harper-subjectivity-rendering/SKILL.md` (create the folder if needed); the agent will pick it up automatically whenever subjectivity or underwriting data could reach a customer-facing message. Teams can also clone this repo and drop the folder in place. The executable ground truth behind the skill (the labeled fixture corpora and the release regression gate) lives in the hercules engineering repo under `tests/eval/`; matcher internals evolve there, while the principles in `SKILL.md` are stable. The `brand-kit/` folder in this repo carries Harper's brand guidelines so any team rendering customer-facing material has the visual standards alongside the copy rules.
